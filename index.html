09/06/2020

Setting up weapon selling!

Game Plan: Finish all 10 waves and set up weapon selling.

How Weapon Selling Works:

  • Player clicks "Sell Weapon" button.
  • Player is told how to sell weapon (click on it), given the option to cancel.
  • When the player clicks, sellingWeapon is checked (true if sell button was clicked).
  • If true, a foreach loop runs through the weapons checking if the clicked element is that weapon.
  • If it is that weapon, it is sold, the loop breaks, and sellingWeapon is set to false.

The Bear enemy was added: medium speed, very high health.

Issues Encountered

Weapon selling is different from weapon placing because it matters what object you click on and which one it is. Weapon placing did not matter where you clicked because it used IsTouching, while this cycles through all weapons to figure out which one it is, took a bit of figuring out.

01/06/2020

Setting up shooting!

Game Plan: Get the bullets positioned properly to align with the weapon image and move in the right direction from there. Set up IsTouching for Bullet x Enemy.

How shooting works:

  • Each tick of the timer a foreach loop checks if the internal counter for shooting for each weapon is ready to shoot.
  • If ready, depending on the type of weapon it creates a bullet.
  • Bullets are given the X/Y positions of the weapon, the speed of the bullet, the damage it does, the angle the weapon is at (rotation), how long the range is, and true/false for the bullet size (true = big bullet).
  • In the bullet class the bullet position is set up depending on the angle (so it shoots from the right side and right spot), and the bullet is created.
  • *Then a foreach loop checks if the bullet is still in it's range and moves it in the correct direction, if the bullet is outside of it's range it deletes itself.
  • Then it is checked if the bullet is still on the screen, if not, it deletes itself.
  • Bullet x Enemy IsTouching is checked, if true, they are both deleted (big bullets are not deleted).
  • From here the loop restarts, if a bullet is already made, it's code to move starts at the *.

14/05/2020

Setting up weapons placing!

Game Plan: When the player buys a weapon they need to place it, but not on top of another object (aka another weapon or the path (to be implemented later).

Adding these to the Weapon Class:

  • public MoveWeapon (used to place the weapon and follow mouse)
  • public IsTouching (check to make sure player does not place overlapping objects)

Creating placeWeaponTimer and bool placingWeapon:

When a weapon is purchased the game is paused and the main timer is stopped. placeWeaponTimer starts and on every tick the position of the mouse is taken and given to the last weapon in the list so it follows the mouse around the screen (but stays in the canvas).

Creating private void gamescreen_click

When the screen is clicked first it is checked if placingWeapon is true. isTouching a new variable, is set to false. Then a foreach loops runs through all of the weapons in the list not including the newest one and checks if it is touching the one being placed using the IsTouching method. If it is touching then the loop breaks and the weapon is not placed. If the foreach loops runs through without returning true (aka is touching something) then the weapon is placed, the placeWeaponTimer is stopped, and the timer is started up again.
